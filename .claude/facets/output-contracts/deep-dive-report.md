# Deep Dive Report Output Contract

**深掘り調査系スキルの出力形式。**

## フォーマット

```markdown
## {調査テーマ}

### 背景

{なぜこの調査が必要か — 1-3文}

### 調査結果

#### {調査観点1}

{詳細な分析結果}

**出典**: {URL / ドキュメント名 / バージョン}

#### {調査観点2}

{詳細な分析結果}

**出典**: {URL / ドキュメント名 / バージョン}

### 分析

{調査結果を横断した考察・パターン・気づき}

### 結論と推奨アクション

1. {推奨アクション1} — 理由: {根拠}
2. {推奨アクション2} — 理由: {根拠}

### 制約・注意事項

- {バージョン依存、環境依存、未検証事項}

### 参考資料

- {出典1}: {URL}
- {出典2}: {URL}
```

## ルール

- 各調査観点には出典を必ず併記する（`factual-writing` Policy 参照）
- 調査日を冒頭または背景セクションに明記する
- 未検証の情報は「推測」と明記する
- 推奨アクションには必ず理由を添える
- 制約・注意事項セクションで情報の限界を明示する
